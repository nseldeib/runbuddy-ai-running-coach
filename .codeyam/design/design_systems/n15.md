# MARGIN №15 · build spec

> An editorial publication that ships like software. One typographic system, one set of tokens, two themes. Apply across whatever surface the brief calls for.

This document is the source of truth for rebuilding the N15 product. Reproduce the feel — the typography, color, components, and patterns — not a fixed page sequence.

## How to use this system

This document describes a design *language*, not a fixed page sequence. It has three layers — apply them in order:

1. **Core language** (always applies) — typography (Playfair Display + Lora + Inter + JetBrains Mono with the strict role split, including italic-word-emphasis in headlines), the closed color palette, spacing, **0px border radius on everything except the mobile phone bezel**, no shadows, no gradients. Every surface inherits these.
2. **Essential components** (use as the brief requires) — surfaces on canvas, basic editorial nav, status pills (DRAFT/EDIT/FINAL/SHIPPED), bracketed IDs (`[01]`, `[a01]`, etc.), diff indicators (`+ 03.v`), simple ink dividers, mode toggle. The everyday vocabulary needed to render almost any page. Bracketed IDs and status pills are part of the brand voice and belong on most product surfaces.
3. **Showcase patterns** (opt-in, only when the brief calls for them) — the iconic flourishes that define this system's voice on hero and marketing surfaces. For Margin, the showcase patterns include: **the massive italic numeral hero (e.g. `N°15` at clamp 100-220px), the system bar with build numbers and live region tag, the editorial three-column nav with central wordmark, marginalia code comments (`// release notes`) positioned absolute around heroes, the lavender (`--haze`) nudge card, the full-bleed black band release-notes section with three-column changelog, the activity log as its own section, and the italic + solar-yellow leading-disc headline treatment.** Do not add these just to "represent" the system; include them only when the brief explicitly asks for that kind of moment.

### Minimum viable composition

For a basic product surface — e.g. a listing page that shows a grid or list of items, each with a few fields — the entire page is:

- A minimal header (small wordmark + basic nav only if the brief implies it; the full system bar with build numbers is showcase, not essential)
- The requested content (a list/table of entries with bracketed IDs, italic titles, status pills, simple metadata) using the standard component recipes
- A filter row only if filtering is implied
- An optional single-line footer

That is the whole page. Do **not** add the massive italic numeral hero, marginalia comments, the lavender nudge card, the black band release-notes section, the activity log as its own section, or any showcase pattern unless the brief explicitly asks for it. The italic-word-in-headline trick and bracketed IDs are part of the core voice and apply to whatever the brief calls for.

### Note on imagery

The "no photographic imagery on the homepage" rule in this doc applies specifically to the typography-as-imagery marketing hero example. **For product surfaces (listings, dashboards, indexes) that need to show photographs of items, render small monochrome thumbnails (~110-120px wide, slight desaturation) inline with each entry.** Small editorial thumbnails fit the magazine register without breaking the system; the rule is specifically against full-bleed photographic heroes.

### Example compositions are examples

Any reference homepage, dashboard, or release-notes pages shown elsewhere in this file are **one** way to compose the system — not the template. **Structure follows the brief, not the examples.** Render only what the brief names; lean on the core language and essential components to give it Margin's voice.

## 1. Product concept

**Margin** is a fictional independent quarterly review of typography and visual culture, published from Buenos Aires and New York. Each issue is treated as a software release.

The voice is **editorial first, technical second**. Big italic Playfair headlines carry the soul. JetBrains Mono carries the data: counts, statuses, version numbers, file paths, timestamps. Lora handles long-form reading. The yellow accent is rare, deliberate, and almost always a leading mark.

## 2. Design tokens

### 2.1 Colors (light, the default)

| Token | Value | Role |
|---|---|---|
| `--canvas` | `#f2f2f2` | Page background, primary surface |
| `--ink` | `#000000` | Primary text, hard borders, heavy elements |
| `--mist` | `#e5e5e5` | Subtle borders, dividers |
| `--haze` | `#cfabef` | Decorative accent fills (sparingly) |
| `--indigo` | `#698fff` | Interactive links, active states, deltas, "next" indicators |
| `--solar` | `#fcf572` | Single accent. Leading marks, highlights, live indicators |

### 2.2 Colors (dark)

Applied via `body.dark { ... }`. Token names stay identical, only values flip.

| Token | Value | Notes |
|---|---|---|
| `--canvas` | `#0e0d0c` | Warm near-black, not pure. Keeps editorial warmth |
| `--ink` | `#ebe7dd` | Warm off-white. Comfortable for long reading |
| `--mist` | `#1f1d1a` | Subtle dark divider |
| `--haze` | `#5a4480` | Deeper lavender. Still recognizable |
| `--indigo` | `#8aa6ff` | Lifted slightly for dark-bg legibility |
| `--solar` | `#fcf572` | Unchanged. The money color works on both |

### 2.3 Color rules

1. The palette is closed. No grays beyond what `color-mix` derives from `--ink` or `--canvas`.
2. For semi-transparent ink (the most common case), use `color-mix(in srgb, var(--ink) X%, transparent)` rather than rgba. This auto-flips with theme.
3. Same for `--canvas` on dark surfaces.
4. The black band (release notes section) intentionally inverts: it uses `background: var(--ink); color: var(--canvas)`. In light mode it reads as a dark inversion, in dark mode it reads as a light inversion. Both work.
5. `--solar` is rare. Used for: the leading dot before headlines, the live indicator, single highlighted phrases inside body copy, the active mode in the toggle. Never as a fill on large areas, never on buttons.
6. `--haze` shows up at most once per surface. It's a nudge, not a brand color.

### 2.4 Typography

| Family | Substitute (Google Fonts) | Role |
|---|---|---|
| Freight Big | **Playfair Display** | Display headlines, big italic moments |
| Freight Text | **Lora** | Long-form body, italic blockquotes |
| Neue Haas Grotesk | **Inter** | UI labels, navigation, small caps |
| Times | **Times New Roman** | Editorial small print, captions |
| (Tech layer) | **JetBrains Mono** | Data: IDs, versions, timestamps, file paths, counts |

### 2.5 Type scale

| Role | Size | Line height | Letter spacing | Family |
|---|---|---|---|---|
| display | 198px (clamp 100-220) | 0.8 | -0.04em | Playfair italic |
| heading-lg | 84-108px | 0.95 | -0.025em | Playfair |
| heading | 56-65px | 0.95 | -0.02em | Playfair italic |
| heading-sm | 38-48px | 1.0 | -0.015em | Playfair |
| sub | 22-28px | 1.1 | -0.01em | Playfair |
| body-lg | 18px | 1.55 | normal | Lora italic |
| body | 14-16px | 1.55-1.65 | normal | Lora |
| ui | 12-13px | 1.4 | 0.02em | Inter |
| label | 10-11px | 1.0 | 0.06-0.12em uppercase | Inter or JetBrains Mono |
| micro | 9-10px | 1.4 | 0.04em | JetBrains Mono |

### 2.6 Spacing scale (px)

`4, 6, 7, 14, 18, 21, 26, 32, 48, 64, 80, 96, 120`

Section gap: 64-96. Card padding: 24-48. Element gap: 12-26. Generous whitespace is the rule.

### 2.7 Borders & radii

- Borders: 1px solid only. Use `--ink` for hard structural top borders, `--mist` for soft dividers.
- Border radius: **0px on everything**. Pills, cards, inputs, buttons, all sharp corners. The phone bezel in mobile is the only exception (42px outer, 32px inner).
- No drop shadows, no elevation, ever.

## 3. Component patterns

### 3.1 System bar

Three columns: left = live status + region + build date, center = small italic "Margin" wordmark, right = version tag + theme toggle + locale.

```
[ • live · margin.press   cdn: bsas-1   build 2026.05.05 ]   [ Margin ]   [ v15.0.0-rc.1   light · dark   ES · EN ]
```

The `•` is solar yellow, pulsing 1.6s.

### 3.2 Editorial nav

Three-column nav below system bar: left = section links, center = wordmark (larger), right = account links. Hover state turns text indigo with a 1px indigo underline.

### 3.3 Hero

- Tiny 3-column metadata strip on top (issue / topic / release date)
- "Marginalia" code-style notes positioned absolute on left and right of hero (`// theme: light · dark`)
- Centered massive italic numeral: `<em>N</em>°15` at clamp(100, 18vw, 220) px. The italic N + a roman ordinal "°15".
- A small solar yellow disc as a leading mark, attached as a `::before` pseudo on the headline so it scales with text. Size: `0.55em` square. Vertical-align: `0.18em`. Margin-right: `0.28em`.
- A monospace version string below: `/margin/issues/15 · the-margin-issue · m15.rc1`
- A "strap" border below with `[ topic ]` label, italic Lora quote in center, build status on right with a small ink dot.

### 3.4 Status pills

Small uppercase Inter or Mono, 1px border. Five states for articles, used identically across all three surfaces:

| State | Label | Style |
|---|---|---|
| Empty | `DRAFT` | Border `--mist`, text 65% ink |
| Working | `EDIT 1`, `EDIT 2` | Border `--mist`, text 70% ink |
| Indigo working | (rare, "in review") | Border `--indigo`, text `--indigo` |
| Final | `FINAL` | Border `--ink`, text `--ink` |
| Shipped | `SHIPPED` | Filled: bg `--ink`, text `--canvas` |

### 3.5 Bracketed IDs

Articles, submissions, release-notes items, and feed entries all carry a JetBrains Mono ID in brackets: `[01]`, `[a01]`, `[c02]`, `[r01]`, `[M-15-241]`. Color: 50% ink. Always 9-11px.

### 3.6 Diff indicators

In the dashboard production table, an "updated" column uses git-log style tokens:
- `+ 03.v` for additions, color `--indigo`
- `~ 02.v` for in-progress, color 55% ink
- `− 28.iv` for removals (rare), color 50% ink

Date format: `dd.[roman month].yyyy`. Example: `28.v.2026`. Roman month is lowercase.

### 3.7 Marginalia / code comments

Tiny mono notes, 10px, 45% ink, prefixed with `//`. Used as system labels (`// release notes`, `// editorial desk`, `// margin: light · dark`). Border-left or border-right 1px mist, padding 10px.

### 3.8 Lavender nudge card

A single `--haze` filled block per surface, used for a one-shot call to action. Big italic headline (one of the words inverted to non-italic via `<em>`), small mono meta line (`// last touched ... · branch ...`), inline CTA button with `--ink` border. CTA hover fills with ink.

### 3.9 Black band

The release-notes section. Full-bleed `--ink` background, `--canvas` text. Three-column changelog grid: Added (+), Changed (~), Removed (−). Each item is a `<li>` with a bracketed id and a Lora line. Footer shows previous / current / next versions, with current highlighted in `--solar` italic.

The big headline inside is `clamp(54px, 7.5vw, 108px)` Playfair, with one word in `--solar` italic ("software"). Leading solar disc on the headline as in 4.3.

### 3.10 Activity log

Border-left 1px mist, padding-left 18px. Each row is a 3-col grid: timestamp (mono, 12px, 45% ink), level tag (10px uppercase, 5 levels: `SHIPPED` indigo, `RESOLVED` neutral, `NEW` ink, `PRESS` neutral, `EXTEND` italic), message (Lora 14px). Inside the message, contributor names use Playfair italic, article titles use Lora italic, and rare highlights use `<span>` with solar background + ink text, padding 0 4px.

Timestamp format: `09:42 -03` (24h with timezone offset). "yesterday" is acceptable for older items.

## 4. Mode toggle

Inline pill in the system bar / page header. Two `<button>` elements: "light" and "dark", separated by a faded `·`. Active button has full opacity and a 1-2px solar underline. JS toggles `body.light` / `body.dark` classes. CSS variables do the rest.

```js
document.querySelectorAll('.mode-toggle button').forEach(btn => {
  btn.addEventListener('click', () => {
    const mode = btn.dataset.mode;
    document.body.classList.remove('light','dark');
    document.body.classList.add(mode);
    document.querySelectorAll('.mode-toggle button')
      .forEach(b => b.classList.toggle('active', b.dataset.mode === mode));
  });
});
```

CSS transition on `body`: `background .35s ease, color .35s ease`. Keep it subtle.

## 5. Voice & copy

### 5.1 Brand vocabulary

The product treats the magazine as software. Use both registers, never apologize for either.

| Editorial term | Tech equivalent (use both) |
|---|---|
| Issue | Release |
| Article | Module / item |
| Edit pass | Version (v1, v2, v3) |
| Goes to press | Ships |
| Cover essay | Hero |
| Submission | PR / queue item |
| Letter to the editor | Comment |

### 5.2 Tone

- Quietly confident. Editorial, not academic. Technical, not corporate.
- Sentences are often short. One full stop, then breathe.
- Italic words in headlines do real work, never decorative. Pick the word that carries the rotation.
- No emoji. No em-dashes (the publication style guide forbids them, replaced with middle dots, commas, or full stops).
- Copy can be slightly self-aware about the magazine-as-software conceit ("A magazine that ships like software"), but only in the release-notes section. Elsewhere, play it straight.

### 5.3 Sample copy that fits

- Hero quote: "On legibility, weight, and the quiet politics of the page."
- Cover essay opener: "For most of typography's history, weight was a question of ink and paper."
- Pull quote: "A heavier letter is not a louder one. It is a letter that has decided to take up more room."
- Lavender nudge: "The cover essay still wants a closing line. Theo, this one is on you."
- Mono comment: `// last touched 11.42 PM · 6 unresolved · branch theo/cover-final`

## 6. Implementation notes

### 6.1 Stack

The HTML files are intentionally vanilla: one `<style>` block per file, one `<script>` block for the toggle, no build step. For a real build, port to:

- **Framework**: React Router 7 or Next.js (depending on the rendering needs)
- **Styling**: CSS variables + utility classes, or Tailwind v4 with the `@theme` block matching section 3
- **Fonts**: Self-hosted via `next/font` or equivalent. Variable Playfair Display + Lora + Inter + JetBrains Mono.
- **Mode persistence**: `localStorage.theme` + initial class set in a no-FOUC inline script in `<head>` before paint.

### 6.2 Tailwind v4 theme block

```css
@theme {
  --color-canvas: #f2f2f2;
  --color-ink: #000000;
  --color-mist: #e5e5e5;
  --color-haze: #cfabef;
  --color-indigo: #698fff;
  --color-solar: #fcf572;

  --font-display: 'Playfair Display', Georgia, serif;
  --font-sans: 'Inter', system-ui, sans-serif;
  --font-mono: 'JetBrains Mono', ui-monospace, monospace;
  --font-serif: 'Lora', 'Times New Roman', serif;
}

@variant dark (&:where(.dark, .dark *));

.dark {
  --color-canvas: #0e0d0c;
  --color-ink: #ebe7dd;
  --color-mist: #1f1d1a;
  --color-haze: #5a4480;
  --color-indigo: #8aa6ff;
  --color-solar: #fcf572;
}
```

### 6.3 Accessibility

- All ink-on-canvas combinations clear AAA at 16px and AA at 12-14px.
- Solar yellow is **not** used for foreground text on canvas in light mode (insufficient contrast). It's only used as a fill behind ink text, or as a small graphic mark, or as foreground on the black band.
- Indigo on canvas passes AA for body sizes; lifted indigo on dark canvas does the same.
- Mode toggle buttons are real `<button>` elements with `data-mode` attributes. Ensure `aria-pressed` reflects state in production.

### 6.4 Don'ts

- No drop shadows, no glassmorphism, no gradients, no glow.
- No filled buttons except the `SHIPPED` status pill and the lavender CTA hover.
- No icons from icon libraries inside body content. The mobile tab bar uses Playfair characters as glyphs. The system bar uses one pulsing dot. That's the visual language.
- No em-dashes anywhere in copy. Use middle dot `·`, comma, or full stop.
- No photographic imagery on the homepage. Type is the imagery.

## 7. Open questions left for you

1. Real font files: Freight Big and Neue Haas Grotesk are the design intent, Playfair / Lora / Inter are the public-domain stand-ins. Decide if the project licenses the originals.
2. Mode default: system-preference (`prefers-color-scheme`) or always-light-on-first-visit.
3. The Playfair italic glyphs (▲ ¶ A ∂ ○) used in some component examples are a stylistic gamble. A real shipping app likely needs proper icons. Decide.
